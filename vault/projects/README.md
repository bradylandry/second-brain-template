# projects/

One file per active project. Work projects, side projects, research — anything you're actively investing time in.

**Filename:** descriptive kebab-case, e.g. `my-cool-app.md`, `q2-migration.md`.

**What each file contains:**
- Problem the project solves
- Solution you're building
- Metrics / impact
- Technical highlights
- Lessons learned
- Dated updates as you work on it (most recent at bottom — see `/braindump` pattern)

**AI integration:** the `/braindump` skill (see `../../integrations/`) is designed to read your freeform session notes, identify which project they belong to, and add a dated section matching this file's house style. Keep project files structured so the AI has something to match.

Use `../_templates/project.md` as the scaffold.
