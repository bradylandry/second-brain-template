# people/

One file per person your AI should know about — colleagues, clients, mentors, family. Optional.

## When to add someone

Create a file when the AI will benefit from context about them recurring. Examples:

- **Your manager** — for prep before 1:1s, the AI should know their style, priorities, history
- **A key client** — so emails to them pick the right tone
- **Your doctor** — when the AI is helping with health tracking / prescriptions
- **Your spouse/kids** — so "pick up X on the way home" lands in the right context

## When NOT to add

- Casual acquaintances — low signal-to-noise
- People you interact with once a year
- Anyone whose context the AI can get from a search (public figures with Wikipedia pages)

## Privacy

People files are personal. If you sync this vault publicly, **`people/` should be in `.gitignore`**. The template does this by default — double-check your setup.

Use `_templates/person.md` as the template for new entries.
