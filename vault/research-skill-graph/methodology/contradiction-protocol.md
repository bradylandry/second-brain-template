# Contradiction Protocol

What to do when sources or lenses disagree. Contradictions are features, not bugs — this is where real insights hide.

## When Two Sources Disagree

### Step 1: Check the basics
- Are they talking about the same thing? (different demographics, time periods, geographies)
- Are they using the same data?
- Is one source a higher tier? (see [[source-evaluation]])

### Step 2: Find the root of disagreement
Usually one of:
- **Different data:** looking at different datasets → find which is more complete/recent
- **Different interpretation:** same data, different conclusions → examine their reasoning
- **Different scope:** one is national, one is Louisiana-specific → both might be right in context
- **Different timeframe:** short-term vs long-term trends can look opposite
- **Different incentives:** one has reason to spin the data → check funding

### Step 3: Document, don't resolve
Write it as:
> "Source A argues [X] based on [data]. Source B argues [Y] based on [data]. The disagreement likely stems from [root cause]. Under conditions [C1], A is probably right. Under conditions [C2], B is probably right."

### Step 4: Upgrade to open-questions if unresolvable
If you can't determine who's right after honest effort:
- "We don't know whether X or Y" IS a finding — valuable intelligence
- Add it to `open-questions.md` with the specific evidence needed to resolve it

## When Two Lenses Disagree

Expected and healthy. Different lenses SHOULD produce tension.

**Confidence adjustment:**
- 2 lenses agree, 1 disagrees → investigate the disagreeing lens deeper. it might see something the others miss
- All lenses agree → be suspicious. you might have confirmation bias. re-read [[../lenses/contrarian]]
- No lenses agree → your research question might be too broad. narrow the scope in the project index
